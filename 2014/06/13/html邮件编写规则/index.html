html邮件编写规则

  • HTML
  • 邮件
WEB

####1. 邮件宽度与高度

邮件宽度最好设置在600~680px之间，过宽的话容易出现横向滚动条，影响用户体验；而高度最好在1024px左右，因为页面过短，传递给客户的信息有限，不容易获得用户信任，而内容过长容易让用户产生厌烦的情绪。

####2. 编码格式

邮件编码格式一般使用utf-8，这样能保障文字在各种客户端上正常显示。

####3.邮件布局

不要使用div+css布局，都使用table来布局和实现，同时在使用table布局的时候不要重复使用colspan，这样容易被邮件客户端判为邮件结构过渡复杂，而归类为垃圾邮件，建议使用table嵌套来实现。

####4. 文本链接规则

1）链接必须使用绝对路径；
2）链接长度最好不要超过256个字符，也不要包含空格、中文字符，否则可能会出现无法打开或者无法跟踪打开率的情况；
3）不要出现文字连接形式，最好采用这种形式，比如Comm100，而不是www.comm100.cn

####5. 图片规则

1）模版尽量不要使用背景图片，背景图片在gmail中默认不展示；
2）图片都要加上相应的alt属性，这样在图片不展示时，也能显示一定信息来；
3）图片的宽度和高度都要指定，不然在图片被拦截是，布局会出现错误；
4）图片控制在8张以内，而且每张图片最大不要超过15kb；
5）图片名称最好不要包含空格、中文字符、或者特殊符号，不然有些邮箱（比如QQ邮箱，gmail邮箱等）可能不能识别出来。

####6. html标签规则

1）尽量不要使用提交表单；
2）padding和margin属性，在table中padding和margin在某些客户端和Web界面（如gmail），中兼容性比较差，尽量少用；如果想留空格，最好使用带有宽度和高度的td来实现；
3）尽量不使用 <h>,<ul>,<p> 这些标签，因为这些标签包含默认样式，容易出现布局混乱，建议能使用td标签都使用td来实现。

####7. 一些增加互动性小技巧

1）适当使用gif图片；
2）使用带有视频样式的图片。

####使用内距padding

当在电子邮件中使用padding，您必须始终指定每一个值(top、right、bottom和left)，否则你可以得到无法预测的结果。我发现你仍然可以使用简写，例如：padding:10px 10px 8px 5px，但是如果你有麻烦，你需要单独写出来，例如：padding-top:10px;padding-right:10px;padding-bottom:8px;padding-left:5px;。

如果你使用padding有更大的麻烦(例如，如果您发送邮件的平台把CSS样式都删除了)，不要使用它。简单的使用空的单元格创建空的间距。没有必要使用空的gif文件到单元格中，只需要确保在单元格中添加行内样式style="line-height:0;font-size:0"，并在单元格中填充一个 并给他设置一个明确的高度或宽度。这里有一个例子：

还请注意，在td标签上使用padding还算是安全的，但在div或p标签上，他们表现得更加不可预测。

因此，我们将使用一些行内CSS给单元格添加内距padding。然后，我们将插入我们的图片，添加alt属性文本和style=”display:block;”，这个是一个常用来修复一些电子邮件客户端图片下面空白间距。我们通过在td标记中添加align=”center”将图像居中显示。我们还将添加一个重要的alt标签，因为我们邮件加载之后，在多数情况下，图片是关闭的。

注意：如果你的头部的内容很重要，在头部中不要仅仅使用一个图像。记住，大多数邮件客户端把图像关闭，所以，要是你的电子邮件很重要，不要使用图片加载进来。在这个例子中，我们的头部是非常多余的。

####创建内容区域

首先，我们将会在中间的单元格中添加一些内距padding，按照我们的设计，表格周围有一些空间。

现在，我们将添加内容，并给中间的单元格添加一些内距(padding)。

现在，我们要在第三行中添加两列内容。因为我们想要在两个单元格之间设置一个margin，但大家都知道表格是不支持margin，我们需要创建一个三列的表格，将一个空的单元格放在两列之间。

尽管我非常喜欢坚持使用百分比，但当你有一个特定大小的内容时，很难将它转换成一个百分比(在这个例子中，列设置为48.1%可能会混淆)。因为这个原因，我们有两个260px宽的图像，我们将创建260px的列，并在两列中间创建一个20px宽的空单元格。(总计是540px，这是因为我们总宽度是600px，减去两边填充的30px内距)。确保在间距的单元格中设置你的字体(font-size)和行高(line-height)为０和插入空白字符 。
