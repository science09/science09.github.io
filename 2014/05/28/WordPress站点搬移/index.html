WordPress站点的搬移

  • WordPress
  • 免费域名
WEB

由于之前申请的域名和空间都是免费的，到了期限，不得不对站点进行搬迁，域名之前是在tk免费域名上进行注册的,该域名只能最多免费一年，一年以后需要续费，否则域名不可使用。

我所使用的tk免费域名已经到了期限，不可再使用，在浏览器中输入www.embfans.tk已经不能正常的呈现出网页内容。因此需要再重新申请域名和购买主机。

  1. 申请新域名和主机
    -

在搬迁之前，需要重新申请域名，否则一切免谈。在域名的申请上，需要花一点时间，考虑域名是否容易记，是否有利于SEO，尽量选择短一点，易于记住的域名。我自己搭建的博客只是平时的一些学习记录，所以还是选择了免费域名，在tk免费域名 注册了一个 www.byte.tk 的域名。关于主机，之前是在DH Mart http://cpanel.dhmart.info/switcher 上注册的账号，该主机可创建多了账户。因此还是在该站点上申请了一个主机空间。重新申请的主机空间后，IP地址和之前的发生了变化，该主机空间又不允许重新设置域名，因此接下来，我们就对站点进行搬移。

  1. 复制所有的文件到本地电脑
  • 使用FTP（FileZilla)工具，登陆到原来站点的空间，把该站点空间根目录下（在我这里是 public_html文件夹下）的所有文件以及文件夹下载下来，把所有文件保存到本地电脑。

  1. 导出数据库
    -
    WordPress内容导出一般可分为三种方法，使用WordPress自带的工具、插件方法和数据库导出。由于我的WordPress站点已经不能访问，所以就只能采用数据库导出的方法，通过数据库导出能够把WordPress所有数据以及设置全部导出，包括插件设置以及主题设置等。通过phpMyAdmin登陆到数据库管理图形界面，对数据库执行导出操作，具体如下图所示。

  2. 创建数据库
    -
    此时需要重新创建一个数据库直接使用cPanel创建数据库，操作简单，输入您要创建的数据库名称以及密码即可。

  3. 修改wp-config.php文件
    -
    现在本地电脑上修改wp-config.php文件，里面有数据库的名称配置和密码配置，以及数据库的主机地址，根据实际情况，修改各个项目的值。修改完毕后，可以把刚才从旧主机站点导出的文件进行压缩，再把整个压缩文件上传到新的主机空间。如果不压缩也可以，但是会延长上传的时间。

  4. 修改数据库中的域名相关信息并上传
    -
    WordPress中的所有文件都上传到新的站点空间后，此时的WordPress是没有数据的，WordPress中的所有文章都保存在数据库中。接下来，说明一下如果原有的数据库中的信息，并上传到新的站点空间。使用文本编辑器，打开导出的sql文件，通过搜索，把原来旧域名的字符串都修改为新的域名，保存文件。

登陆到新的主机站点空间，打开phpMyAdmin，如果里面有内容，先把原来的数据删除干净，再通道导入按钮导入sql文件。导入完毕后刷新，就可以看到我们上传的数据库信息。

  1. 访问新域名
    -
    在浏览器中输入新的域名 www.byte.tk 查看是否能正常呈现网页内容。 检查一下网站上的菜单以及文章标题，查看是否能够正常跳转，如果不能正常跳转，可能是数据中还包含老域名的字符串，需要对数据库进行修改。最后再登陆后台，查看各个功能是否正常。

  2. 修域名做重定向
    -
    显然，我们不希望流量流失，所以你要引导你的老域名访问到新域名。有两种方法可以实现：
    第一就是在老主机设置301定向，一般的主机都有;
    另一种就是修改.htaccess文件，在老主机根目录下，打开，删除所有数据，添加下面的内容：
    redirect 301 / 新域名地址

通过以上几个步骤，就完成了WordPress站点的完美搬迁。
